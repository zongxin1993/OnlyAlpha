# MyQuant 到 OnlyAlpha 迁移规范

## 1. 原工程

```text
/home/zongxin/workspace/MyQuant
```

MyQuant 是可运行系统，OnlyAlpha 必须将其视为行为基线，而不是只把它视为代码来源。

## 2. 迁移前分析

必须建立：

```text
docs/myquant_analysis.md
```

至少记录：

- 目录结构；
- 主入口；
- 实盘入口；
- 回测入口；
- 策略接口；
- 行情流；
- 下单流；
- 委托回报；
- 成交回报；
- 持仓更新；
- 账户更新；
- 缓存；
- 数据库；
- 配置；
- 日志；
- 并发；
- 外部依赖；
- 已知问题；
- 可复用能力；
- 不建议迁移的实现。

## 3. 模块映射

必须维护：

| MyQuant 模块 | 当前职责 | 当前问题 | OnlyAlpha 模块 | 迁移状态 |
|---|---|---|---|---|

## 4. 迁移方式

允许：

- 重新实现；
- 小范围封装；
- 适配器；
- 并行运行；
- 双写；
- 回归对比。

不允许：

- 整目录复制；
- 批量改名；
- 在新架构中永久保留旧全局状态；
- 通过兼容代码掩盖架构错误。

## 5. 回归原则

从 MyQuant 选取固定策略和固定历史数据，比较：

- 信号时间；
- 委托方向；
- 委托价格；
- 成交结果；
- 持仓变化；
- 手续费；
- 滑点；
- 收益；
- 最大回撤；
- 异常行为。

差异必须可解释并记录。

9. 投研；
10. Web；
11. 策略迁移；
